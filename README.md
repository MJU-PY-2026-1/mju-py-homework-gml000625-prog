## 2. 시나리오 (5~10줄)
* 웨이퍼 결함 분석 관리 시스템은 반도체 공정에서 웨이퍼의 결함 정보를 입력하고 상태를 확인할 수 있는 콘솔 프로그램입니다. 사용자는 웨이퍼 번호, 공정명, 결함 개수, 결함 면적, 결함 위치를 입력하여 각 웨이퍼의 상태를 간단하게 관리할 수 있습니다. 프로그램은 입력된 정보를 바탕으로 결함 점수를 계산하고 웨이퍼를 정상, 주의, 위험 상태로 구분합니다. 이를 통해 사용자는 어떤 웨이퍼에 문제가 있는지 빠르게 확인할 수 있습니다. 이후 단계에서는 여러 웨이퍼의 정보를 리스트에 저장하여 전체 목록을 출력하고 평균 결함 점수나 가장 위험한 웨이퍼를 비교할 수 있도록 확장할 예정입니다. 또한 메뉴 기능을 추가하여 등록, 조회, 분석, 검색을 반복적으로 수행할 수 있게 만들 계획입니다. 최종적으로는 함수와 딕셔너리를 활용하여 웨이퍼 정보를 체계적으로 저장하고 관리하는 반도체 공정 데이터 관리 프로그램으로 발전시키고자 합니다.

🛠️ Troubleshooting & 기술 회고

문제 1 : 주제가 너무 어려워질 가능성

원인: 처음에는 반도체 전공과 연결하려는 생각이 커서 실제 공정 분석처럼 복잡한 프로그램으로 가려는 방향이 있었음.
해결: 웨이퍼 결함을 복잡하게 해석하는 대신 결함 정보를 입력하고 상태를 판정하는 관리 시스템 형태로 단순화함. 전공성과 과제 난이도를 모두 맞출 수 있도록 조정함.

문제 2 : 1차 과제에 비해 기능이 과도하게 많아질 위험

원인: 처음부터 메뉴, 검색, 통계, 저장 기능까지 한 번에 넣으려 하면 코드가 길어지고 1차 과제 핵심 문법이 흐려질 수 있었음.
해결: 기능을 1차, 2차, 3차, 4차로 나누어 단계적으로 발전시키는 구조로 정리함. 1차는 입력과 조건문 중심으로 제한하고 이후 단계에서 리스트, 메뉴, 함수, 딕셔너리를 추가하기로 함.

문제 3 : 변수 이름이 모호하면 코드 이해가 어려움

원인: 짧고 단순한 변수명을 사용하면 나중에 코드가 길어질수록 의미 파악이 어려워짐.
해결: wafer_id, defect_count, defect_area, defect_location, total_score, grade처럼 의미가 분명한 변수명으로 다시 설계함. 가독성과 확장성을 함께 확보함.

문제 4 : 자료형 구분이 명확하지 않으면 계산이 불편함

원인: 입력값을 모두 문자열로 처리하면 결함 개수나 결함 면적을 계산할 때 형변환 문제가 생길 수 있음.
해결: 문자열, 정수, 실수, 논리형을 구분해서 설계함. 입력값의 역할에 따라 자료형을 명확히 나누어 계산과 판정이 자연스럽게 이루어지도록 함.

문제 5 : 결함 점수 계산식이 너무 복잡해질 수 있음

원인: 실제 반도체 공정처럼 복잡한 가중치 계산식을 넣으면 초급 파이썬 과제 수준에서 구현과 설명이 어려워짐.
해결: 결함 개수, 결함 면적, 결함 위치를 각각 점수화한 뒤 합산하는 단순한 방식으로 변경함. 조건문으로 구현하기 쉽고 결과 설명도 쉬운 구조로 정리함.

문제 6 : 프로그램이 계산기처럼 보일 수 있음

원인: 웨이퍼 1개만 입력받고 결과를 출력한 뒤 종료되는 구조는 관리 시스템보다는 단순 계산기 느낌이 강함.
해결: 여러 웨이퍼 정보를 리스트에 저장하고 메뉴를 통해 등록, 조회, 분석, 검색이 가능하도록 확장 방향을 설계함. 이를 통해 데이터 관리 시스템처럼 보이도록 개선함.

문제 7 : 저장 구조를 어떻게 잡을지 애매했음

원인: 웨이퍼 하나에 여러 정보가 들어가므로 어떤 방식으로 저장해야 나중에 출력과 검색이 쉬울지 고민이 있었음.
해결: 초기에는 같은 순서의 리스트 여러 개를 사용하는 방식으로 단순화하고 최종 단계에서는 리스트 안에 딕셔너리를 저장하는 방향으로 확장하기로 정리함. 초급 수준과 확장성을 모두 고려한 선택이었음.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 특별 칭호를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 용사의 이름, 체력, 마력, 공격력을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 종합 전투력 산출: (체력 * 0.7) + (마력 * 0.5) + (공격력 * 1.8).
  * if-elif-else를 사용하여 S등급부터 F등급까지 5단계 등급 판정 로직 추가.
  * and 연산자를 활용하여 '전설의 대마법사' 등 특별 칭호 부여 기능 구현.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
